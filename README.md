# jc-portfolio
All about John's work

## Project Structure

This portfolio is maintained using a documentation-first workflow.

### Homepage

The homepage introduces my background, skills, experience, and featured projects.

### Project Case Studies

Every featured project is documented in two formats:

* **Markdown (`.md`)** — Source of truth used for writing and maintaining project documentation.
* **HTML (`.html`)** — Public presentation generated from the reusable project template.

### Workflow

1. Create a Markdown document.
2. Document the project.
3. Copy `_TEMPLATE.html`.
4. Generate the HTML page from the Markdown using GitHub Copilot.
5. Link the project from the homepage.

This workflow keeps project documentation easy to maintain while ensuring a consistent presentation across the portfolio.
